## 1. Контекст проекта

### 1.1 Бизнес-задача

**Проблема:**  
Врачи-терапевты тратят до 40% времени приёма на заполнение медицинской документации. Это снижает качество внимания к пациенту и ведёт к выгоранию персонала. Даже если есть текстовый транскрипт (например, аудиозаписи приёма или онлайн-чата), его необходимо вручную сокращать и форматировать.

**Решение:**  
  
Наш NLP-сервис принимает на вход «грязный» текст диалога и преобразует его в лаконичный медицинский протокол по стандарту SOAP (Subjective, Objective, Assessment, Plan).

**Критерии успеха:**

1. Корректное распределение фактов по разделам  
   (жалобы — в *S*, осмотр — в *O*).

2. Отсутствие «галлюцинаций»  
   (модель не должна придумывать симптомы, которых нет в тексте).

3. Использование правильной медицинской терминологии  
   (например, замена «красное горло» на «гиперемия зева»).

### 1.2 Целевая аудитория и пользователи

**Пользователи:** Терапевты, врачи телемедицинских сервисов.

**Сценарии:** Врач вставляет текст диалога (из чата или готового транскрипта) в окно сервиса и получает структурированную заметку.

**Нагрузка:** Низкая (обработка текста — быстрая операция).

### 1.3 Ограничения и допущения

**Ограничение:** Мы работаем только с текстом. Обработка аудио не входит в рамки проекта.

**Допущение:** Входной текст содержит достаточно информации для заполнения хотя бы 3-х разделов SOAP.

## 2. Архитектура решения

### 2.1 Общая схема

Text Input (Dialog) -> Preprocessing -> LLM (Summarizer/Extractor) -> Post-processing -> SOAP Note.

**Core:** LLM (например, Llama 3 или Saiga), настроенная через Few-shot промптинг.

**Post-processing:** Проверка на стоп-слова и форматирование.

### 2.2 Хранилище знаний

**Источники:** Для MVP база знаний не требуется, вся информация берется из входного текста.

### 2.3 Интеграции

**UI:** Веб-интерфейс (Streamlit или Gradio)
